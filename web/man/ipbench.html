NAME

ipbench - Control an ipbench session  

SYNTAX

ipbench --client=client_hostname... [--client-port=port]
--test=testname --test-target=test_target_hostname
[--test-args=args] [--test-port=port]
[--target=target_hostname] [--target-port=port]
[--target-args=args]
[-v] [-V] [-h|--help]  

DESCRIPTION

Controls a number of remote testing clients to run a specified test and report their results.  

OPTIONS

--client=hostname...
Specify a remote client to connect to. Each client must be running an invocation of ipbenchd(1). You may specify multiple clients by passing the --client argument many times.
--client-port=port
Specify a port to connect to remote client. This will be global, i.e. all clients will be contacted on this port.
--test=name
The test to run. Use -h to see a list of possible tests.
--test-args=arguments
Arguments for the test. See test specific documentation for possible arguments; by leaving this argument out you will get the test defaults.
--test-target=hostname
The target machine that the clients should direct their testing towards.
--test-port=port
The port to run the test on. Each test has an inbuilt port it will use by default.
--target=target_hostname
If the test has companion test that is run on the target machine, this flag will activate it. The target machine must be running ipbenchtd(1). See the test documentation for more information about the companion target test.
--target-args=args
Arguments for the companion test on the target.
-v
Be more verbose.
-V
Show version information.
-h | --help
Show help information.
 

FILES

(none)  

ENVIRONMENT VARIABLES

(none)  

EXAMPLES

The latency test might be run as:

ipbench --client=tinny0 --client=tinny1 --client-port=6123
--target=leffe --active-target --target-port=6124
--target-args="warmup=10,cooldown=10"
--test=latency --test-port=5555
--test-args="bps=100000000,size=128,socktype=udp,warmup=10,cooldown=10"
